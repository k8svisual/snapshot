VpK Snapshot

VpK (Visual parsed Kubernetes) Snapshot retrieves information from a kubernetes (k8s) instance and creates a snapshot directory with a separate file for each unique k8s resource. The k8s resource type must support the get verb to be included in the snapshot.

View screen image below the following field numbers for additional assistance.

  • (1)Enter the command this is used to communicate with kubernetes to retrieve the resource information. The default is kubectl. Other command examples: oc, microk8 kubectl

  • (2)Number of seconds to wait for command to execute before the command is terminated.

  • (3)A default value of 'snapshot' will be appended to the front of the snapshot directory that is created in the output root directory. The default value will be replaced with this value if provided.
    Example directory name: snapshot-2021-11-17-16h-11m-38s

  • (4)Output root directory is the location where the new snapshot directory will be created. To modify the provided value press the 'CHANGE OUTPUT DIRECTORY' button. A system dialog to select a directory will be displayed.

  • (5)Execution information will be displayed in this area. All of the commands that are executed along with status data will be shown. If any errors occur a summary of the data will be included in this area. This is a scrollable and selectable area that can be copied to the clip board.

  • (6)Press the 'GET SNAPSHOT' button to begin the snapshot capature.

During the capture process messages will scroll in the status area. Once completed a message in a green bar will be shown at the bottom of the screen indicating the resource files have been created. The status area will also have a similar message along with the snapshot directory that caontains the files.
